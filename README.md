# WeekTrackerBot

Персональный Telegram-бот для фиксации рабочих достижений. Напоминает записать итоги периода, сохраняет ответы и автоматически формирует сводки с помощью локальной LLM. Поддерживает нескольких пользователей с индивидуальными расписаниями и часовыми поясами.

## Возможности

- Регистрация пользователя с выбором часового пояса.
- Настройка напоминаний:
  - с выбором конкретных дней недели (только будни, произвольный набор);
  - еженедельно в определённый день;
  - ежемесячно в указанное число.
- Автоматические отчёты через локальную LLM (Ollama):
  - еженедельные или ежемесячные;
  - структурированное резюме с разделами: проекты, решённые проблемы, полученные навыки;
  - сохранение отчётов в базе с возможностью просмотра истории.
- Режим отпуска (`/vacation N`) – временное отключение напоминаний.
- Команды для администратора:
  - просмотр времени системы;
  - поставленные задачи;
  - разовая рассылка сообщений всем активным пользователям (`/broadcast`), в том числе автоматическая при обновлении версии.
- Устойчивость к перезапускам: все напоминания и автоотчёты восстанавливаются.
- Логирование в файл с ротацией и в консоль.
- Миграции базы данных без потери данных.

## Планы по доработке

- Веб-интерфейс для просмотра статистики.
- Экспорт записей в PDF/CSV.
- Поддержка нескольких LLM-провайдеров.
- Более детальная аналитика по тегам или проектам.

## Технологии

- Python 3.10+
- python-telegram-bot (с `job-queue` для планирования)
- aiosqlite (асинхронная работа с SQLite)
- Ollama (локальный запуск LLM)
- pytz (часовые пояса)
- python-dotenv (конфигурация через `.env`)

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <url-репозитория>
cd weektrackerbot
```

2. Виртуальное окружение
```bash
python -m venv venv
source venv/bin/activate      # Linux/macOS
venv\Scripts\activate         # Windows
```

3. Установка зависимостей
```bash
pip install -r requirements.txt
```

4. Конфигурация .env
Создайте в корне файл .env со следующим содержимым:

```text
BOT_TOKEN=<ваш_токен_от_BotFather>
DB_NAME=tracker.db
LOG_LEVEL=INFO
ADMIN_IDS=123456789
MODEL_NAME=qwen2.5:0.5b-instruct-q4_K_M
BOT_TOKEN — токен Telegram-бота.
```
`ADMIN_IDS` — ID администраторов через запятую.

`MODEL_NAME` — имя модели Ollama (должна быть предварительно загружена).

5. Миграция базы данных
```bash
python migrate.py
```
6. Запуск бота
```bash
python bot.py
```
Запуск тестов
```bash
pytest tests/ -v
```
Docker
```Dockerfile
dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "bot.py"]
```
docker-compose.yml (опционально)
```yaml
version: '3.8'
services:
  bot:
    build: .
    env_file:
      - .env
    volumes:
      - ./tracker.db:/app/tracker.db
      - ./bot.log:/app/bot.log
```
Перед запуском в Docker также необходимо выполнить миграцию: `docker compose run bot python migrate.py.`

Важные замечания
Перед первым запуском убедитесь, что сервер Ollama работает и модель загружена (ollama pull <MODEL_NAME>).

При обновлении версии может потребоваться повторная миграция (python migrate.py) — она безопасна и не затронет существующие данные.

Логи записываются в bot.log, уровень детализации настраивается через `LOG_LEVEL`.
